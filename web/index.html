What is EcoScale?

EcoScale is a Carbon-Aware Kubernetes Controller — a piece of software that runs inside your cluster and makes recommendations based on real-time carbon intensity (CO2 per kWh) of the cloud region powering your nodes.

Unlike the default Kubernetes scheduler (which only cares about CPU and RAM), EcoScale understands that different regions have different carbon footprints. It suggests when to scale down, drain nodes, or shift capacity to greener regions — all without increasing your cloud bill.

Why EcoScale?

Cloud emissions are a growing concern. A single data center can consume as much power as a small city. The grid mix — how much comes from solar, wind, hydro vs coal and gas — varies by region and time of day.

The Problem

Kubernetes has no concept of carbon. It schedules pods wherever there's capacity, regardless of whether that region runs on coal or hydro.

The Opportunity

us-west-2 (Oregon) is ~2–3× greener than us-east-1 (Virginia). Shifting non-critical workloads can cut emissions 20–30% without extra cost.

How Does It Work?

One Helm chart. One label. That's it.

  1. 1
    Install — Deploy EcoScale to your cluster with Helm in the kube-system namespace.
  2. 2
    Label — Add ecoscale/flexible: "true" to workloads that can be rescheduled.
  3. 3
    Observe — EcoScale fetches carbon intensity, compares regions, and outputs recommendations via API and Prometheus metrics.

One Label. That's It.

Add a single label to any Deployment or Pod. EcoScale handles the rest.

batch-job.yaml 
# Mark this workload as carbon-flexible
# EcoScale will consider it for Sun-Chaser scaling
apiVersion: apps/v1
kind: Deployment
metadata:
  name: batch-processor
spec:
  template:
    metadata:
      labels:
        app: batch-processor
        ecoscale/flexible: "true"  # ← That's all you need
    spec:
      containers:
        - name: processor
          image: myapp:latest
